Guide de mise à jour du site GitHub

Instructions détaillées pour déployer et mettre à jour votre site

Suivez ce guide étape par étape pour mettre à jour facilement votre site sur GitHub Pages

Déploiement initial sur GitHub Pages

1
Créer un nouveau dépôt sur GitHub
  1. Connectez-vous à votre compte GitHub
  2. Cliquez sur le bouton "+" en haut à droite, puis sélectionnez "New repository"
  3. Nommez votre dépôt (par exemple : ia-medecine)
  4. Choisissez l'option "Public" pour que le site soit accessible à tous
  5. Ne cochez pas l'option "Initialize this repository with a README"
  6. Cliquez sur "Create repository"
Astuce : Si vous souhaitez que votre site soit accessible à l'adresse username.github.io (sans sous-dossier), nommez votre dépôt exactement username.github.io (où username est votre nom d'utilisateur GitHub).
2
Télécharger les fichiers du site web sur GitHub

Option 1 : Via l'interface web de GitHub

  1. Sur la page de votre nouveau dépôt, cliquez sur "uploading an existing file"
  2. Décompressez le fichier ia_medecine_site.zip sur votre ordinateur
  3. Faites glisser tous les fichiers et dossiers du site web dans la zone de dépôt de GitHub
  4. Ajoutez un message de commit comme "Initial commit - Site web IA en médecine"
  5. Cliquez sur "Commit changes"

Option 2 : Via Git en ligne de commande (pour utilisateurs avancés)

  1. Décompressez le fichier ia_medecine_site.zip sur votre ordinateur
  2. Ouvrez un terminal et naviguez vers le dossier contenant les fichiers du site
  3. Exécutez les commandes suivantes :
git init
git add .
git commit -m "Initial commit - Site web IA en médecine"
git branch -M main
git remote add origin https://github.com/VOTRE-NOM-UTILISATEUR/ia-medecine.git
git push -u origin main
Note : Remplacez VOTRE-NOM-UTILISATEUR par votre nom d'utilisateur GitHub et ia-medecine par le nom de votre dépôt si différent.
3
Activer GitHub Pages
  1. Sur la page de votre dépôt, cliquez sur "Settings" (onglet avec l'icône d'engrenage)
  2. Dans le menu de gauche, cliquez sur "Pages" (sous la section "Code and automation")
  3. Dans la section "Source", sélectionnez "Deploy from a branch"
  4. Dans la section "Branch", sélectionnez "main" et "/root", puis cliquez sur "Save"
  5. Attendez quelques minutes que GitHub déploie votre site
  6. Rafraîchissez la page, vous verrez un message indiquant "Your site is published at https://VOTRE-NOM-UTILISATEUR.github.io/ia-medecine/"
Attention : Le déploiement peut prendre jusqu'à 10 minutes. Si votre site n'est pas accessible après ce délai, vérifiez que la structure de vos fichiers est correcte et que le fichier index.html est bien à la racine du dépôt.
4
Vérifier le déploiement
  1. Cliquez sur le lien fourni par GitHub (https://VOTRE-NOM-UTILISATEUR.github.io/ia-medecine/)
  2. Vérifiez que toutes les pages et fonctionnalités du site fonctionnent correctement
  3. Testez la navigation et l'affichage sur différents appareils (ordinateur, tablette, smartphone)
Astuce : Utilisez les outils de développement de votre navigateur (F12) pour vérifier s'il y a des erreurs dans la console qui pourraient indiquer des problèmes avec votre site.

Mettre à jour votre site

1
Télécharger les fichiers mis à jour

Option 1 : Via l'interface web de GitHub

  1. Naviguez vers votre dépôt GitHub
  2. Pour chaque fichier que vous souhaitez mettre à jour :
    • Cliquez sur le fichier pour l'ouvrir
    • Cliquez sur l'icône de crayon (Edit this file) en haut à droite
    • Effectuez vos modifications
    • Faites défiler jusqu'en bas et ajoutez un message de commit
    • Cliquez sur "Commit changes"
  3. Pour ajouter de nouveaux fichiers, cliquez sur "Add file" puis "Upload files" depuis la page principale du dépôt

Option 2 : Via Git en ligne de commande (recommandé pour les mises à jour importantes)

  1. Clonez d'abord votre dépôt si ce n'est pas déjà fait :
git clone https://github.com/VOTRE-NOM-UTILISATEUR/ia-medecine.git
cd ia-medecine
  1. Effectuez vos modifications localement
  2. Ajoutez, committez et poussez vos changements :
git add .
git commit -m "Mise à jour du site - Description des changements"
git push origin main
2
Mettre à jour la présentation PDF
  1. Assurez-vous que votre nouvelle présentation PowerPoint est convertie en PDF
  2. Remplacez le fichier PDF existant dans le dossier pdf de votre dépôt
  3. Si vous utilisez l'interface web de GitHub :
    • Naviguez vers le dossier pdf
    • Cliquez sur le fichier PDF existant
    • Cliquez sur "Delete this file" (icône de corbeille)
    • Committez la suppression
    • Retournez au dossier pdf et cliquez sur "Add file" puis "Upload files"
    • Téléversez votre nouveau fichier PDF
    • Committez l'ajout
Astuce : Pour maintenir la compatibilité des liens, assurez-vous que le nouveau fichier PDF a exactement le même nom que l'ancien.
3
Ajouter la page sur la réglementation de l'IA en Afrique
  1. Assurez-vous que le fichier reglementation.html est bien présent à la racine de votre dépôt
  2. Vérifiez que les liens vers cette page sont correctement configurés dans les autres pages du site
  3. Si vous utilisez l'interface web de GitHub, téléversez le fichier comme expliqué précédemment
Note : Si vous avez déjà une page sur la réglementation mais souhaitez la mettre à jour, suivez les instructions de mise à jour de fichiers ci-dessus.
4
Vérifier les mises à jour
  1. Attendez quelques minutes que GitHub Pages redéploie votre site
  2. Visitez votre site à l'adresse https://VOTRE-NOM-UTILISATEUR.github.io/ia-medecine/
  3. Vérifiez que toutes les mises à jour apparaissent correctement
  4. Testez les nouveaux liens et fonctionnalités
  5. Vérifiez que la présentation PDF mise à jour est accessible
Attention : Si les mises à jour n'apparaissent pas, essayez de vider le cache de votre navigateur ou d'ouvrir le site en mode navigation privée.

Résolution des problèmes courants

Images manquantes

Si certaines images ne s'affichent pas sur votre site déployé :

  1. Vérifiez que les chemins des images dans votre code HTML sont relatifs et corrects
  2. Assurez-vous que les images sont bien présentes dans le dépôt GitHub
  3. Vérifiez que la casse (majuscules/minuscules) des noms de fichiers correspond exactement
<img src="images/logo.png" alt="Logo">
<img src="/images/logo.png" alt="Logo">

Problèmes de CSS

Si le style de votre site ne s'affiche pas correctement :

  1. Vérifiez que les liens vers les fichiers CSS sont relatifs
  2. Assurez-vous que les fichiers CSS sont bien présents dans le dépôt GitHub
  3. Inspectez la console du navigateur pour voir s'il y a des erreurs 404 pour les fichiers CSS

Site non mis à jour

Si votre site ne reflète pas les dernières modifications :

  1. Attendez quelques minutes (jusqu'à 10 minutes) pour que GitHub Pages redéploie votre site
  2. Videz le cache de votre navigateur (Ctrl+F5 ou Cmd+Shift+R)
  3. Vérifiez dans l'onglet "Actions" de votre dépôt GitHub que le déploiement a bien été effectué

Erreur 404 - Page non trouvée

Si vous obtenez une erreur 404 en accédant à votre site :

  1. Vérifiez que GitHub Pages est bien activé dans les paramètres de votre dépôt
  2. Assurez-vous que le fichier index.html est bien à la racine du dépôt
  3. Vérifiez que l'URL que vous utilisez est correcte (attention à la casse)
